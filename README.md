# 📝 프로젝트 #3
### 📝 1. 프로젝트 제목 :
   회의록 관리 게시판 (개인 프로젝트)

### 📝 2. 프로젝트 개발 기간 :
   2026.04.13 ~ 2026.04.17

### 📝 3. 프로젝트 개요 : 
   회의록 데이터(일정,  안건,  참여자,  회의실) 를 관리하는 프로그램

### 📝 4. 프로젝트 선정 이유 : 
   도커(Docker)를 써보고 싶었습니다. 직접 세팅한 AWS Lightsail(Ubuntu22)에서 Docker를 경험해 봤습니다.


### ✨ 5. 주요 기능
* **회의록 관리:** 목록 조회, 상세 보기, 검색(JPA 페이징 처리)
* **참석자 관리:** 회의별 참석자 등록 및 수정 기능
* **파일 시스템:** 첨부파일 업로드(미리보기 지원) 및 다운로드
* **데이터 보안:** 논리 삭제(Soft Delete)를 통한 데이터 보존성 확보

### 🛠 6. 기술 스택 (Tech Stack)
* **Backend:** Java 17, Spring Boot, JPA, MariaDB, Lombok, Swagger
* * **Frontend:** HTML/CSS, JavaScript, jQuery, Ajax, Bootstrap
* **DevOps:** AWS Lightsail, Docker

### 🛠 7. 성장점과 느낀점 :
* 왜 다들 도커 도커 하는지 알 것 같습니다. Docker를 쓰면 컨테이너에 담아주는 등 컨테이너만 관리하면 다 된다는 점이 수동으로 .war를 배포하는 과정에 비해 간편해서 좋았습니다.
* 실무에서 많이 사용하는 기술은,  그 이유가 있는 것 같습니다. 더 많은 기술을 경험해 보고 싶습니다.
